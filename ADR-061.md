---

## 1. Context

Helios 平台 22 章 spec 已完成全量重写。在 BTC 板块（§4 + §5 + §6）已跑通的当前状态下，识别到一个**根本性工程问题**：

22 章业务模块中存在大量**算法操作的概念性重复**——同一个数学操作（e.g. bootstrap CI、Z-score 标准化、Wasserstein 距离、regime filter）会在多个章节里独立实现。

具体扫描结果（基于 22 章 spec 文本）：

```yaml
duplicate_observations:
  ops.zscore_normalize:    7 章使用 (§4 / §5 / §6 / §7 / §15 / §18 / §19)
  ops.regime_filter_data:  7 章使用 (§5 / §13 / §17 / §18 / §11 / §15 / §6)
  ops.bootstrap_ci:        7 章使用 (§9 / §13 / §18 / §20 / §17 / §8 / §6)
  ops.drawdown_curve:      8 章使用 (§5 / §9 / §11 / §13 / §17 / §15 / §18 / §6)
  ops.sharpe_ratio:        7 章使用 (§9 / §11 / §13 / §17 / §6 / §18 / 多处)

如果 22 章独立实现:
  9 板块 × 31 个 atomic ops = 279 处重复实现位置
  bug 表面积: 279 个独立 bug 候选位置
  代码量: 重复 30-40%
  测试覆盖: 难达标 (业务模块测试侧重业务逻辑)
```

如果不抽公共库，**每个板块新增** = 重复实现 31 个 op 一遍，**bug 表面积乘以 9 倍**。

## 2. Decision

建立 Helios 元实现层 (`helios.ops`)：

- **Layer 1 (本 ADR)**：31 个**原子元实现** (atomic ops)
  - 严格定义：能独立完成一项任务、不依赖其它元实现
  - 依赖：仅 Layer 0 标准库（numpy / scipy / pandas / sklearn）
  - 内部互不调用

- **Layer 2 (后续 ADR-062)**：约 8-10 个**元 skill** (复合操作)
  - 元实现的组合（e.g. PSR/DSR = bootstrap_ci + skew_kurt + N_eff 估计）
  - 依赖：仅 Layer 1（不调其它 Layer 2）
  - 待 Layer 1 完工后启动

- **Layer 3 (业务模块, 已存在)**：22 章
  - 自由调用 Layer 1 + Layer 2

### 2.2 三大架构纪律（必须永不破例）

```yaml
discipline_1_layer_1_no_internal_call:
  规则: Layer 1 元实现 内部禁止互相调用
  实施: lint check, helios.ops 内部禁止 import 自身其它模块
  违反案例: 若发现 ops.psr_dsr 内部调用 ops.bootstrap_ci → 移到 Layer 2
  例外: 无

discipline_2_layer_2_no_internal_call:
  规则: Layer 2 元 skill 之间禁止互相调用, 仅调 Layer 1
  实施: lint check, helios.skills 内部禁止 import 自身其它模块
  违反案例: 若发现 cpcv_full_pipeline 调 walk_forward_split → 重新组合 Layer 1

discipline_3_business_must_use_layer_1_2:
  规则: 22 章业务模块禁止重新实现 Layer 1/2 已有功能
  实施: code review + lint rule
  违反案例: §13 自己写 bootstrap → reject PR
  例外: 性能临界路径可以自实现, 但需要单独 ADR 论证
```

## 7. 风险与对冲

```yaml
risks:

  risk_1_premature_abstraction:
    描述: 现在仅 BTC 跑通, 其它 22 章未实现, 抽象可能不准
    severity: low
    mitigation:
      - 本 ADR 基于 22 章 spec 系统扫描, 不是凭空设计
      - 31 op 全部有具体 spec 引用
      - 启动后若发现 ambiguity, 单独 op 可暂缓

  risk_2_cc_implementation_quality:
    描述: 多 CC 并行可能产生 quality 不一致
    severity: medium
    mitigation:
      - 每个 op 一份独立 prompt (产物 2)
      - 强制学术对照测试
      - Wiki review PR + merge
      - lint check + benchmark check + 测试覆盖率门槛

  risk_3_layer_3_business_modules_bypass_ops:
    描述: 业务模块可能图省事自己实现
    severity: medium
    mitigation:
      - 三大架构纪律 + lint rule
      - code review 关注点
      - 性能临界路径要单独 ADR 论证

  risk_4_third_party_alternative_emerges:
    描述: 抽出来后, scipy/statsmodels 可能后续提供同样能力
    severity: low
    mitigation:
      - 学术对照测试 + 性能基准
      - 每年 review 一次, 第三方稳定 → 切换 (helios.ops 改为 wrapper)

  risk_5_pydantic_overhead:
    描述: 性能敏感 op 用 Pydantic 输入验证可能太慢
    severity: low
    mitigation:
      - 性能敏感 op 用 dataclass + manual validation (e.g. dtw_distance)
      - 默认 Pydantic, 性能不达标 → 切 dataclass

  risk_6_breaking_existing_btc:
    描述: BTC 板块 retrofit 时可能 break
    severity: low (因 retrofit 不强制)
    mitigation:
      - retrofit 时跑 BTC regression test
      - retrofit 一个 op 一个 PR, 可单独 rollback
```

## 8. Alternatives Considered

```yaml
alternative_1_no_atomic_ops_layer:
  approach: 22 章各自实现, 不抽公共库
  reason_rejected:
    - 9 板块复制时 279 处重复实现
    - bug 表面积乘以 9
    - 测试覆盖率难达标
    - 9 板块完工后再抽, 重构成本是 9x

alternative_2_thin_wrapper_only:
  approach: 仅做 scipy/numpy thin wrapper, 不写自实现
  reason_rejected:
    - 多个 op 没有现成实现 (e.g. brier_decomposed / purge_embargo / regime_*)
    - thin wrapper 没有真正解决"接口一致性"问题
    - 仍然每章独立实现剩下的部分

alternative_3_in_22_chapter_layer:
  approach: 每章节内自包含 utils, 不跨章共享
  reason_rejected:
    - 跨章 utils 才是真重复源头
    - in-chapter utils 不能解决 cross-chapter 问题

alternative_4_skip_layer_2_meta_skills:
  approach: 仅做 Layer 1, 不做 Layer 2
  reason_rejected:
    - PSR/DSR/CPCV 这种复合操作仍然每章实现一次
    - Layer 2 是 Layer 1 之上的自然抽象
    - 但 Layer 2 推迟到 Layer 1 完工后是 OK 的 (本 ADR 决定)

alternative_5_external_library:
  approach: 用现有 quant library (e.g. zipline / qstrader / vectorbt)
  reason_rejected:
    - 这些库 too opinionated, 与 Helios 哲学不匹配
    - License 问题
    - 集成成本可能 > 自实现
```
